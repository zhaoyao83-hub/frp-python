# MyFRP Web 管理面板 - 技术规格说明文档

> **实现状态说明**：本文档描述的独立 Web 管理面板**已完整实现**。
>
> 当前项目包含两套 Dashboard：
> - **frps.py 内嵌简易 API**：配置 `webapi_port` 后，服务端在 `handle_webapi()` 中提供 `GET /` 和 `GET /stats`，返回 `Stats.get_all_stats()` 的 JSON 数据。无鉴权、无 WebSocket、无前端页面。作为独立 Dashboard 的 stats 数据源使用。frps.json 中 `webapi_port` 已调整为 7501 以避让独立 Dashboard 的 7500 端口。
> - **独立 Web 管理面板（manage.py）**：基于 FastAPI + React + JWT + WebSocket 的完整管理系统，已实现本文档描述的全部功能（鉴权、配置管理、服务启停、实时监控、代理管理）。入口为 `python manage.py -c config/webadmin.json`，默认监听 7500 端口。
>
> 本文档下述内容为独立 Web 面板的完整设计规格，与实际实现高度吻合。

## 1. 概述

### 1.1 目标
为 MyFRP 提供一个独立部署的 Web 管理面板，实现：
- **仪表盘**：服务运行状态概览
- **配置管理**：在线编辑 frps.json / frpc.json
- **服务启停**：启动/停止/重启 frps 与 frpc 进程
- **运行状态监控**：代理、连接、流量、日志实时监控

### 1.2 技术栈
| 层 | 选型 | 说明 |
|----|------|------|
| 前端 | React 18 + Vite + Ant Design 5 | SPA，构建产物由后端托管 |
| 后端 | FastAPI (Python 3.8+) | 原生 asyncio，与项目一致 |
| 鉴权 | JWT (PyJWT) + 密码哈希 (passlib[bcrypt]) | name/pwd 登录 |
| 进程管理 | asyncio.subprocess | 启停 frps/frpc 子进程 |
| 实时通信 | WebSocket | 日志/状态推送 |
| 部署 | 独立进程（manage.py） | 不嵌入 frps.py |

### 1.3 部署架构

## 2. 鉴权设计

### 2.1 用户存储
- 用户凭据存储于 `config/webadmin.json` 配置文件（首次启动若不存在则生成默认 admin 用户）
- 密码使用 bcrypt 哈希存储

**webadmin.json 结构**：
```json
{
  "host": "0.0.0.0",
  "port": 7500,
  "secret_key": "<auto-generated-uuid>",
  "token_expire_minutes": 720,
  "users": [
    {
      "username": "admin",
      "password_hash": "$2b$12$...",
      "role": "admin"
    }
  ],
  "frps_config_path": "config/frps.json",
  "frpc_config_path": "config/frpc.json",
  "log_buffer_lines": 1000
}
```

### 2.2 认证流程
1. 用户提交 `username` + `password` → `POST /api/auth/login`
2. 后端校验 bcrypt 哈希，签发 JWT（claims: `sub`, `role`, `exp`）
3. 前端存储 access_token（localStorage），后续请求带 `Authorization: Bearer <token>`
4. WebSocket 通过 query param 传递 token：`/ws/logs?token=<jwt>`
5. Token 过期后调用 `POST /api/auth/refresh` 刷新

### 2.3 权限
- **admin**：全部功能（启停服务、修改配置、管理用户）
- **viewer**（预留）：仅查看仪表盘/监控，不可修改

### 2.4 安全措施
- JWT secret_key 自动生成并持久化到 config/webadmin.json
- 密码最小长度 8 位
- 登录失败 5 次锁定 5 分钟（内存计数器）
- 生产建议：dashboard 监听 `127.0.0.1` 或加 TLS（复用 server.crt）

## 3. 后端 API 设计

### 3.1 鉴权 API

| 方法 | 路径 | 说明 | 请求体 | 响应 |
|------|------|------|--------|------|
| POST | `/api/auth/login` | 登录 | `{username, password}` | `{access_token, token_type, expires_in}` |
| POST | `/api/auth/logout` | 登出（前端清 token） | - | `{message}` |
| POST | `/api/auth/refresh` | 刷新 token | - | `{access_token, expires_in}` |
| GET | `/api/auth/me` | 当前用户信息 | - | `{username, role}` |
| POST | `/api/auth/users` | 创建用户（admin） | `{username, password, role}` | `{username, role}` |
| GET | `/api/auth/users` | 用户列表（admin） | - | `[{username, role}]` |
| DELETE | `/api/auth/users/{username}` | 删除用户（admin） | - | `{message}` |
| PUT | `/api/auth/password` | 修改密码 | `{old_password, new_password}` | `{message}` |

### 3.2 仪表盘 API

| 方法 | 路径 | 说明 | 响应字段 |
|------|------|------|----------|
| GET | `/api/dashboard/overview` | 总览 | `frps_status, frpc_status, uptime, total_proxies, current_connections, total_bytes_in, total_bytes_out` |
| GET | `/api/dashboard/proxies` | 代理列表 | `[{name, type, remote_port, status, current_conns, total_conns, bytes_in, bytes_out, created_at}]` |
| GET | `/api/dashboard/connections` | 活跃连接 | `[{conn_id, proxy_name, bytes_in, bytes_out, created_at}]` |

**数据来源**：dashboard 通过 HTTP 请求 frps 内置的 `/stats` 端点（已有 `handle_webapi`）获取 Stats 数据。frpc 状态通过进程存活判断。

### 3.3 配置管理 API

| 方法 | 路径 | 说明 | 请求/响应 |
|------|------|------|-----------|
| GET | `/api/config/{type}` | 读取配置（type=server/client） | 响应：配置 JSON + schema |
| PUT | `/api/config/{type}` | 保存配置 | 请求：配置 JSON；响应：`{message, need_restart}` |
| POST | `/api/config/{type}/validate` | 校验配置（不保存） | 请求：配置 JSON；响应：`{valid, errors[]}` |
| GET | `/api/config/{type}/schema` | 获取配置字段 schema | 响应：字段定义（类型/默认/必填/说明） |
| GET | `/api/config/{type}/raw` | 原始文本读取 | 响应：`{content}` |
| PUT | `/api/config/{type}/raw` | 原始文本保存 | 请求：`{content}`；响应：`{message, need_restart}` |

**校验规则**（服务端配置）：
- `bind_port`：1-65535，必填
- `vhost_http_port`：1-65535 或 null，HTTP 虚拟主机监听端口
- `subdomain_host`：字符串或 null，子域名根域名（如 `example.com`）
- `auth_token`：可选，字符串
- `max_connections`：>=1
- `idle_timeout`：>=10
- `tls`：bool
- `webapi_port`：1-65535 或 null
- `allow_ports`：格式 `"8080,9000-9100"` 或 null
- `ip_whitelist`/`ip_blacklist`：IP/CIDR 逗号分隔或 null

**校验规则**（客户端配置）：
- `server_addr`：有效 IP/域名
- `server_port`：1-65535
- `proxies[]`：每项 `name` 唯一非空，按类型校验必填字段：
  - **tcp/udp**：`local_port` + `remote_port`
  - **http**：`local_port` + `custom_domains` 或 `subdomain` 至少一个
  - **stcp**（提供方）：`local_port` + `sk`
  - **stcp_visitor**（访问方）：`sk` + `server_name` + `bind_port`

**配置变更感知**：保存配置后返回 `need_restart=true`（因为 frps/frpc 启动时加载配置，运行时不支持热更新）。

### 3.4 服务启停 API

| 方法 | 路径 | 说明 | 响应 |
|------|------|------|------|
| GET | `/api/service/{name}/status` | 服务状态（name=frps/frpc） | `{name, running, pid, uptime, restart_count}` |
| POST | `/api/service/{name}/start` | 启动服务 | `{name, pid, message}` |
| POST | `/api/service/{name}/stop` | 停止服务 | `{name, message}` |
| POST | `/api/service/{name}/restart` | 重启服务 | `{name, pid, message}` |

**实现细节**：
- 使用 `asyncio.create_subprocess_exec` 启动 `python frps.py -c config/frps.json`
- 子进程 stdout/stderr 通过 asyncio stream 实时读取，写入内存环形缓冲（默认 1000 行）
- 记录 `pid`、`start_time`、`restart_count`
- 子进程异常退出时记录退出码与原因
- dashboard 自身退出时优雅终止所有子进程（信号处理）

**状态机**：
```
stopped --start--> running --stop--> stopped
   |                  |
   |                  | crash
   |                  v
   +<--auto-restart<--exited
```
- `auto_restart`：可选配置项，默认 false；开启后异常退出自动拉起

### 3.5 监控 API

| 方法 | 路径 | 说明 | 响应 |
|------|------|------|------|
| GET | `/api/monitor/stats` | 实时统计快照 | frps /stats 返回内容 |
| GET | `/api/monitor/logs/{name}` | 历史日志（name=frps/frpc/dashboard） | `{lines: [...], truncated}` |
| GET | `/api/monitor/logs/{name}/download` | 下载完整日志 | text/plain |
| DELETE | `/api/monitor/logs/{name}` | 清空日志缓冲 | `{message}` |

### 3.6 WebSocket API

| 路径 | 说明 | 推送内容 |
|------|------|----------|
| `/ws/logs?token=<jwt>&name=frps` | 实时日志流 | `{ts, level, name, message}` |
| `/ws/logs?token=<jwt>&name=frpc` | 实时日志流 | 同上 |
| `/ws/stats?token=<jwt>` | 实时统计 | 每 2s 推送一次 `/api/monitor/stats` 内容 |

**鉴权**：WebSocket 连接时校验 query param 中的 JWT，无效则关闭连接（code=4401）。

## 6. 关键实现要点

### 6.1 子进程管理（service_manager.py）

```python
class ServiceManager:
    def __init__(self, name, cmd_args, config_path, auto_restart=False):
        self.name = name
        self.cmd_args = cmd_args  # ["python", "frps.py", "-c", "config/frps.json"]
        self.process: Optional[asyncio.subprocess.Process] = None
        self.start_time = None
        self.restart_count = 0
        self.auto_restart = auto_restart
        self.log_buffer = LogBuffer(1000)

    async def start(self) -> dict:
        if self.process and self.process.returncode is None:
            raise RuntimeError(f"{self.name} already running")
        self.process = await asyncio.create_subprocess_exec(
            *self.cmd_args,
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.STDOUT,
            cwd=PROJECT_ROOT,
        )
        self.start_time = time.time()
        asyncio.create_task(self._read_output())
        asyncio.create_task(self._wait_exit())
        return {"name": self.name, "pid": self.process.pid}

    async def stop(self, timeout=5) -> dict:
        if not self.process:
            return {"name": self.name, "message": "not running"}
        self.auto_restart = False  # 主动停止不自动拉起
        self.process.terminate()
        try:
            await asyncio.wait_for(self.process.wait(), timeout)
        except asyncio.TimeoutError:
            self.process.kill()
        return {"name": self.name, "message": "stopped"}

    async def _wait_exit(self):
        await self.process.wait()
        exit_code = self.process.returncode
        self.log_buffer.append(f"[{self.name}] exited code={exit_code}")
        if self.auto_restart and exit_code != 0:
            self.restart_count += 1
            await asyncio.sleep(1)
            await self.start()
```

### 6.2 日志环形缓冲（log_buffer.py）

```python
from collections import deque

class LogBuffer:
    def __init__(self, max_lines=1000):
        self._buffer = deque(maxlen=max_lines)
        self._subscribers = set()

    def append(self, line: str):
        self._buffer.append(line)
        for queue in list(self._subscribers):
            queue.put_nowait(line)

    def get_history(self) -> list:
        return list(self._buffer)

    async def subscribe(self):
        queue = asyncio.Queue()
        self._subscribers.add(queue)
        try:
            while True:
                line = await queue.get()
                yield line
        finally:
            self._subscribers.discard(queue)
```

## 10. 风险与约束

1. **子进程管理**：dashboard 异常退出可能遗留 frps/frpc 孤儿进程；通过 signal handler 优雅终止
2. **跨平台**：Windows 下信号处理与子进程行为略有差异，优先保证 macOS/Linux
3. **单线程约束**：dashboard 与 frps/frpc 是独立进程，互不影响；dashboard 内部全 asyncio
4. **配置热更新**：frps/frpc 不支持运行时热加载，配置变更需重启服务（UI 明确提示）
5. **日志来源**：frps/frpc 子进程的 stdout 由 dashboard 捕获；若 frps 直接写文件则 dashboard 无法获取（需统一走 stdout）

## 11. P3 功能管理（HTTP 代理 & STCP）

Web 管理面板完整支持 P3 阶段新增的 HTTP 虚拟主机代理和 STCP 秘密 TCP 代理的配置管理。

### 11.1 HTTP 代理配置

**服务端**（服务端配置页 → 表单模式）：
- `vhost_http_port`：HTTP 虚拟主机监听端口（如 8080）
- `subdomain_host`：子域名根域名（如 `example.com`）

**客户端**（端口映射页 → 新增/编辑 → 类型选「HTTP 虚拟主机」）：
- `custom_domains`：自定义域名列表，逗号分隔（如 `app.example.com,www.example.com`）
- `subdomain`：子域名前缀（如 `web`，配合服务端 subdomain_host 使用）
- 至少填写 `custom_domains` 或 `subdomain` 其中之一

### 11.2 STCP 代理配置

STCP 分为「提供方」和「访问方」两种角色，均在端口映射页管理：

**STCP 提供方**（type = stcp）：
- 运行本地服务的机器使用此类型
- 必填：`local_port`（本地服务端口）、`sk`（共享密钥）

**STCP 访问方**（type = stcp_visitor）：
- 要访问服务的机器使用此类型
- 必填：`sk`（共享密钥，与提供方一致）、`server_name`（提供方的代理名称）、`bind_port`（本地监听端口）
- 可选：`bind_addr`（本地监听地址，默认 127.0.0.1）

### 11.3 端口映射页功能

- 列表展示：名称、类型、本地服务、映射信息（按类型显示域名/密钥/监听地址）
- 新增/编辑：根据代理类型动态显示表单字段
- 启用/禁用：一键切换代理状态
- 删除：带二次确认
- 配置保存后需重启 frpc 生效（UI 明确提示）
