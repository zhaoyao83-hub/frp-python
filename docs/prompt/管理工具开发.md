### M002. 修复 Dashboard 端口冲突

**原始提示词**：
> （用户反馈）访问不了管理界面

**提示词摘要**：
用户反馈打不开管理面板，排查发现是 frps 内置的统计接口（`dashboard_port`）与独立 Dashboard 服务都占用了 7500 端口，导致 frps 先启动占了端口，Dashboard UI 无法访问。

**根因分析**：
```
frps.json 里配置了 dashboard_port: 7500
    ↓
frps 启动时监听 7500 端口，提供 /stats 接口
    ↓
dashboard.py 再启动 7500 端口失败（或用户访问时看到的是 /stats 数据）
    ↓
管理面板 UI 无法访问
```

**解决方案**：
- `frps.json` 的 `dashboard_port` 改为 **7501**（作为内部统计 API 端口）
- `dashboard.py` 继续使用 **7500**（作为管理面板 Web UI 端口）
- 管理面板后端通过 7501 端口调用 frps 的统计 API

**产出**：
- 修复 `frps.json` 配置：`dashboard_port: 7501`
- 访问 http://localhost:7500 正常进入管理面板登录页

**经验教训**：
- FRP 内置的统计端口（dashboard_port / webapi_port）和独立管理面板是两个不同的东西
- 命名容易混淆，后续需要做命名区分

### M005. 目录重构：web/ + webadmin/ → webadmin/api + webadmin/app

**原始提示词**：
> 把目录 'web' 和 'webadmin' 改个名字，更直观好读

**提示词摘要**：
之前的目录结构比较混乱（有 `web/` 和 `webadmin/` 两个目录），用户要求重新组织，让结构更清晰。

**讨论过程**：
经过讨论确定了最终结构：
```
webadmin/           ← 管理面板总目录
├── api/            ← 后端 FastAPI 代码（原 webadmin/ 的 Python 文件）
├── app/            ← 前端 React 代码（原 web/ 的前端文件）
└── manage.py       ← 启动入口（原 webadmin.py，改名避免与目录重名冲突）
```

**产出**：
- 目录重命名与重组：
  - `webadmin/`（Python 文件）→ `webadmin/api/`
  - `web/`（前端文件）→ `webadmin/app/`
  - `webadmin.py` → `manage.py`（启动入口）
- 更新所有导入路径：
  - `webadmin/api/app.py`
  - `service_manager.py`
  - `config_manager.py`
- 更新静态文件路径：`web/dist` → `webadmin/app/dist`

**关键决策**：
- 入口文件叫 `manage.py` 而不是 `webadmin.py`，因为 Python 包不能和模块同名（`import webadmin` 会有歧义）

**提示词摘要**：

**产出（后端 API）**：
- `webadmin/api/schemas.py` — 数据模型扩展：
  - `ProxyConfig` 新增字段：`sk` / `server_name` / `bind_addr` / `bind_port`
  - `local_port` / `remote_port` 改为可选（不同类型需要不同字段）
  - `ProxyUpdateRequest` 同步更新
- `webadmin/api/routes/proxies.py` — 代理类型校验升级：
  - 支持类型从 tcp/udp 扩展到 tcp/udp/http/stcp/stcp_visitor
  - 按类型差异化校验必填字段
    - tcp/udp/ftp：需要 `local_port` + `remote_port`
    - http：需要 `custom_domains` 或 `subdomain`
    - stcp（提供方）：需要 `local_port` + `sk`
    - stcp_visitor（访问方）：需要 `sk` + `server_name` + `bind_port`
- `webadmin/api/config_manager.py` — 服务端配置扩展：
  - 新增 `vhost_http_port` / `subdomain_host` 字段及端口范围校验
  - 客户端 proxies 的 `item_schema` 完善全部字段
  - 全局校验升级为类型感知

**产出（前端 UI）**：
- `webadmin/app/src/api/proxies.ts` — TypeScript 类型扩展：
  - 导出 `ProxyType` 联合类型
  - 新增 `enabled` / `sk` / `server_name` / `bind_addr` / `bind_port` 字段
- `webadmin/app/src/pages/PortMapping.tsx` — 端口映射页全面升级：
  - 代理类型下拉扩展为 5 种（带中文描述）
  - **条件表单**：用 `Form.useWatch` 监听 type 字段，动态显示/隐藏字段
    - HTTP 类型：显示自定义域名（逗号分隔转数组）、子域名前缀
    - STCP 提供方：显示密钥 SK
    - STCP 访问方：显示密钥、提供方名称、本地监听地址/端口
  - 列表「映射信息」列按类型展示关键信息

**关键技术点**：
- 条件渲染用 `Form.useWatch` 比 state 更可靠（直接从表单状态派生）
- 后端做类型差异化校验，前端做类型差异化展示，前后端对齐

### M010. 服务状态与遗留进程处理

**原始提示词**：
> 两个问题：服务状态显示不一致、禁用的映射规则不生效

**提示词摘要**：
用户反馈两个运维相关的问题，需要修复：

**问题 1：服务状态显示不一致**

根因：管理面板检测不到之前从命令行手动启动的 frps/frpc 进程，导致显示"未运行"但实际端口被占用。

解决方案：
- `service_manager.py` 增加遗留进程检测：
  - `_find_external_processes()` — 通过端口扫描查找外部启动的进程
  - `start(kill_existing=True)` — 启动时可选择先杀掉遗留进程
  - `kill_external()` — 清理遗留进程的 API
- `service.py` — 新增相关 API 端点
- 前端 `StatusCard.tsx` — 显示警告信息和「清理并启动」按钮

**问题 2：禁用的映射规则不生效**

根因：
1. 后端 API 校验缺少 ftp、https 等类型，导致保存失败
2. 前端开关有竞态条件：切换后马上刷新列表，API 请求还没回来就被旧数据覆盖了
3. frpc 需要重启才能加载新配置

解决方案：
- 后端 `proxies.py`：`_VALID_TYPES` 补充全部代理类型
- 后端 `config_manager.py`：`valid_types` 同步补充
- 前端 `PortMapping.tsx`：
  - 增加 `toggleLoading` 状态
  - 切换成功后立即更新本地状态（不等列表刷新）
  - Switch 组件加 loading 状态
  - 增加「立即重启 frpc」按钮，方便配置生效

## 第六阶段：安全与稳定性（2026-07-11）

### M012. WebAPI 端口无认证安全风险修复

**原始提示词**：
> （修复）WebAPI 端口无认证的高危安全问题，避免情报泄露

**提示词摘要**：
frps 的 webapi 端口（7501）之前绑定在 `0.0.0.0`，公网可以直接访问 `/stats` 接口获取流量、连接数等敏感信息，存在信息泄露风险。

**解决方案**：
- `frps.json` 中 `webapi_addr` 从 `0.0.0.0` 改为 `127.0.0.1`
- 只允许本机（管理面板后端）访问，公网无法直接访问
- 管理面板通过 `127.0.0.1:7501` 调用 frps 统计 API

**安全设计原则**：
- 内部 API 只监听回环地址
- 公网只暴露管理面板的 7500 端口（有登录认证）
- 遵循最小暴露原则

### 技术栈

| 层级 | 技术 |
|------|------|
| 后端框架 | FastAPI (Python) |
| 前端框架 | React 18 + TypeScript |
| 构建工具 | Vite |
| UI 组件库 | Ant Design |
| 状态管理 | Zustand（轻量） |

| 实时通信 | WebSocket（日志、监控） |
| 进程管理 | Python subprocess + psutil |

### 关键开发经验

| 经验 | 说明 |
|------|------|
| **命名要提前规划好** | dashboard / webapi / webadmin 三个名字容易混淆，早期花了不少时间做命名清理 |
| **入口文件别和目录同名** | Python 的 `import webadmin` 会有歧义，所以入口叫 `manage.py` |
| **条件表单用 useWatch** | 动态表单用 `Form.useWatch` 比自己维护 state 更可靠 |
| **开关要防竞态** | 切换状态后立即刷新列表会有竞态，应该先更新本地状态 |
| **内部 API 绑回环地址** | frps webapi 只绑 127.0.0.1，公网只暴露有认证的面板端口 |
| **遗留进程检测很重要** | 用户可能从命令行启动过服务，管理面板要能检测到并处理 |
| **配置改动需要重启生效** | frps/frpc 不支持热更新，UI 要明确提示用户 |
| **httpx 要禁用系统代理** | `trust_env=False`，否则本地 API 调用可能被代理拦截 |
