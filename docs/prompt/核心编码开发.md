# MyFRP 核心编码开发 - 提示词复盘

> 本文档按时间顺序整理 MyFRP 项目开发过程中的关键提示词（用户指令）及其产出摘要，用于复盘整个开发历程。

## 一、项目初始化 & 最小可用版本

### P001. 项目起步：实现一个基于 Python asyncio 的反向代理工具

**提示词摘要**：
> 实现一个类似 frp 的轻量级反向代理工具，使用 Python asyncio，支持 TCP 和 UDP 端口转发。要求有服务端和客户端，配置文件驱动，心跳保活。

**产出**：
- `frps.py` — 服务端主程序，监听控制端口，注册代理，转发数据
- `frpc.py` — 客户端主程序，连接服务端，注册本地服务
- `config.py` — 配置加载模块（JSON）
- `protocol.py` — 消息协议（JSON 帧格式：4 字节长度 + JSON payload）
- `log.py` — 日志模块
- `config/frps.json`、`config/frpc.json` — 示例配置
- `README.md` — 项目说明

## 三、P1 阶段：生产级增强

### P003. P1 生产级增强：可安全上线

**提示词摘要**：
> 实现 P1 阶段目标：TLS 加密、访问控制、监控统计、优雅关闭。使系统达到生产级标准，可安全上线运营。参考 p1_enterprise.md 设计文档。

**产出**：
- **TLS 加密**：控制通道与数据通道的 TLS 加密传输
  - 服务端：`tls` / `tls_cert_file` / `tls_key_file`
  - 客户端：`tls` / `tls_insecure` / `tls_ca_file`
- **访问控制**：
  - `ip_whitelist` / `ip_blacklist` — IP 黑白名单（CIDR 支持）
  - `allow_ports` — 允许的远程端口范围
  - `access.py` — AccessControl 类
- **监控统计**：
  - frps 内置 WebAPI：`webapi_port` / `webapi_addr`
  - `/stats` 端点返回流量、连接数等指标
  - `stats.py` — Stats 统计模块
- **优雅关闭**：SIGINT/SIGTERM 信号处理，平滑关闭连接

## 四、P2 阶段：高性能增强

### P004. P2 高性能增强：支撑高并发

**提示词摘要**：
> 实现 P2 阶段目标：二进制协议、数据通道分离、零拷贝优化。使系统支撑高并发场景。参考 p2_high_performance.md 设计文档。

**产出**：
- **二进制协议**（8 字节帧头）：
  - Magic(1B) + Version(1B) + Type(1B) + Flags(1B) + Length(4B BE)
  - DATA 消息：16 字节二进制 UUID + 原始数据（零拷贝）
  - 控制消息：JSON UTF-8
  - `protocol.py` 中 `_TYPE_TO_CODE` 映射表
- **数据通道分离**：
  - 独立 `data_port` 配置，控制/数据通道分离
  - `DATA_AUTH` / `DATA_AUTH_RESP` 握手（携带 session_id）
  - 会话复用模式（向后兼容，不配置 data_port 则复用控制通道）
- **零拷贝优化**：
  - bytearray 预分配编码缓冲
  - 64KB 读缓冲区
  - TCP_NODELAY 设置
  - 256KB socket 发送/接收缓冲

**性能提升**：DATA 消息体积减少 72%，localhost 吞吐量 ~243 MB/s

## 六、P3 阶段：功能完善（HTTP 代理 & STCP）

### P006. 实现 P3 功能完善

**提示词摘要**：
> 实现 P3 阶段目标：HTTP 代理（Host 路由）、STCP/XTCP 秘密 TCP 代理，达到媲美 frp 的功能完整性。

**产出**：
- **HTTP 虚拟主机代理**：
  - 服务端：`vhost_http_port` 监听端口，解析 HTTP 请求 Host 头路由
  - 支持 `custom_domains`（自定义域名）和 `subdomain`（子域名）两种方式
  - 服务端 `subdomain_host` 配置项
  - 协议新增 `HTTP_NEW_CONN` / `HTTP_RESP_REQ` 消息类型
- **STCP（Secret TCP）秘密 TCP**：
  - 提供方（provider）：注册本地服务，设置密钥 sk
  - 访问方（visitor）：通过密钥认证连接，本地监听端口
  - 无需公网端口，点对点 TCP 穿透
  - 协议新增 `STCP_REGISTER` / `STCP_VISITOR_REGISTER` / `STCP_VISITOR_REGISTER_RESP` / `STCP_NEW_VISITOR` / `STCP_VISITOR_READY`
  - 服务端数据缓冲机制（访问者数据在提供方就绪前不丢失）
- `test_p3.py` — HTTP 和 STCP 集成测试
- 更新 `protocol.py`、`frps.py`、`frpc.py`、`config.py`

## 八、提示词使用最佳实践（复盘总结）

### 8.1 高效开发模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **设计文档先行** | 先写 `pX_xxx.md` 设计文档，再按文档实现 | 大型功能阶段（P0/P1/P2/P3） |
| **探索 → 规划 → 执行** | 先用搜索/读取工具摸清架构，再建 TODO 列表，最后动手改 | 对现有代码做增量修改 |
| **文档一致性检查** | 功能实现后专门跑一轮文档同步 | 每个阶段收尾时 |
| **前后端并行** | 后端 schemas + 前端 types 同步更新 | 涉及 API 变更时 |

### 8.2 常见坑点 & 避坑指南

1. **协议变更要同步**：`protocol.py` 的消息类型和编码映射要一起改，文档也要更新
2. **配置字段三处对齐**：`config.py` 默认值 + `config_manager.py` schema + 前端 types，三处要一致
3. **STCP 角色别搞混**：provider（提供方）和 visitor（访问方）是两种代理类型，字段和校验都不同
4. **表单条件渲染**：前端用 `Form.useWatch` 监听 type 字段动态显隐，比 state 更可靠
5. **配置修改需重启**：frps/frpc 不支持热更新，保存后要明确提示用户重启服务

### 8.3 关键文件索引

| 功能域 | 后端文件 | 前端文件 | 文档 |
|--------|----------|----------|------|
| 协议定义 | `protocol.py` | — | `docs/design.md` |
| 服务端核心 | `frps.py` | — | `docs/connection_flow.md` |
| 客户端核心 | `frpc.py` | — | `docs/new_conn_detail.md` |
| 配置管理 | `config.py` + `webadmin/api/config_manager.py` | `webadmin/app/src/components/ConfigEditor.tsx` | `README.md` 配置说明 |
